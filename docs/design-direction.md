# Typing Level Zero デザイン方向性

この文書は、機能仕様とは別に、デザインの大まかな方向性だけを共有するための簡易ブリーフである。具体的なレイアウト、フォント、コンポーネント、装飾はデザイナーの提案に委ねる。

## 共通方針

- 白・黒・グレーを中心にした、シンプルでかっこいい見た目にする。
- 派手な色や過度なゲーム感は避ける。
- メイン画面と分析画面で情報量にメリハリをつける。
- 余白とタイポグラフィを活かし、安っぽく見えないようにする。

## メインプレイページ

- 10文字のpromptを主役にする。
- 入力中は、できるだけ画面上の情報を少なくする。
- 入力済み、現在入力する文字、未入力の文字が自然に分かるようにする。
- スコアや細かい統計はプレイ画面に詰め込まない。

## 全体分析ページ `/analysis`

- メインページより詳しく、練習結果を振り返れる画面にする。
- 時間、速度、正答率、ミス、play履歴などを見やすく表示する。
- 情報量は多くてよいが、雑然とした管理画面にはしない。

## キー別分析ページ `/keys`

- 3ページの中で最もデザインに個性を出す。
- キーボードやキーキャップを視覚的な主役にする。
- 苦手キー・得意キーなどのランキングを、見た瞬間に理解できる形で表示する。
- 各キーについて、どのキーと間違えたか、入力速度や正答率などを詳しく見られるようにする。

## デザイナーへの依頼

メインプレイページ、`/analysis`、`/keys`の3画面について、白黒を基調にした統一感のあるデザイン案を作成してください。メインは静かで集中できる画面、分析ページは情報を詳しく見られる画面、キー別分析ページはキーボードとランキングが印象に残る画面を希望します。具体的な表現方法は自由に提案してください。

機能やデータの詳細は`docs/spec.md`を参照してください。
